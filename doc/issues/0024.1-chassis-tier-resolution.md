## Context

0024 shipped `SubscribeForm` with prop `tiers: Tier[]` where each tier is a full object including a hardcoded `price_display: "$9.99/month"` string. Project layer supplies the array; chassis renders it. This works but has three sharp edges:

1. **Drift risk = trust violation.** Project hardcodes `"$9.99/month"`; admin updates Stripe Dashboard Price to $14.99 without redeploying frontend; users see $9.99 but get charged $14.99. Highest-severity outcome of the whole subscribe surface.
2. **Two-edit maintenance burden.** Every price change requires Stripe Dashboard edit + frontend code change.
3. **Duplication of source-of-truth.** Stripe Price already has `unit_amount`, `currency`, `recurring.interval`, `recurring.interval_count`. Stripe Product has `description`. The chassis can read all of these and format display itself.

This ticket flips the contract: **project layer supplies only the list of `lookup_keys` it wants to surface; chassis pulls everything else from Stripe at request time** and formats display via a chassis helper. Stripe becomes the single source of truth for tier pricing/copy.

After 0024.1, project-layer config drops from ~5 fields × N tiers to 1 string × N tiers. Updating prices, names, or descriptions is a Stripe Dashboard edit only — no code change.

## Design decisions (pre-committed)

1. **New endpoint `GET /billing/tiers?lookup_keys=...`** — comma-separated lookup_keys query param. Returns enriched envelopes pulled from Stripe Price + Product. Empty/absent param returns `[]` (chassis no-op for projects that don't surface subscriptions).

2. **Auth: authed-only** (`Depends(get_current_user)`). Not verified-gated — chassis subscribe page is reachable for unverified users (matches change-email's design from 0017). The endpoint exposes price metadata; not particularly sensitive but no reason to expose to unauthenticated callers in the chassis default. Adopters wanting a public pricing page proxy through their own endpoint.

3. **Stripe API call shape:** single `stripe.Price.list(lookup_keys=lookup_keys, active=True, expand=["data.product"], limit=100)`. The `expand=["data.product"]` pulls Product description in one round-trip — no N+1.

4. **Per-Price metadata required** (chassis-discipline contract from 0024 + payments.md §Subscription tier contract):
   - `metadata.tier_name` — display name. Required.
   - `metadata.grant_micros` — period grant in micros. Required.
   - `Product.description` — marketing copy. Optional (renders empty/null if absent).

5. **Missing metadata = soft skip with warning.** If a returned Price lacks `tier_name` or `grant_micros`, the endpoint logs a structured warning and excludes the tier from the response. Rationale: a tier with missing metadata can't be subscribed to anyway (`POST /billing/subscribe` 422s on missing keys per 0024); skipping at the list endpoint prevents broken UI affordances. Endpoint never returns a half-populated tier.

6. **Currency handling — USD-only per chassis** (`payments.md §Currency`, `BILLING_CURRENCY=usd` locked for v1). If a returned Price's `currency != "usd"`, log warning and still return the tier with `$` prefix. Multi-currency is a future chassis extension, not 0024.1 scope.

7. **New chassis helper `app.billing.format_price`** — sits alongside `format_balance` from 0023. Signature:
   ```python
   def format_price(amount_cents: int, currency: str, interval: str, interval_count: int = 1) -> str:
       """Display Stripe Price as '$9.99/month' / '$99/year' / '$15 every 3 months'."""
   ```
   - Whole dollars render without decimal: `$10/month` not `$10.00/month`.
   - Sub-dollar uses 2 decimals: `$0.99/month`.
   - `interval_count > 1` switches to `"every N intervals"` form.
   - Currency != USD logs warning + uses `$` prefix as fallback.

8. **Frontend `useTiers(lookupKeys: string[])` hook** — reuses chassis lib/api + React Query. Stable cache key via sorted lookup_keys. `staleTime: 5min` (tiers change rarely; frequent re-fetches are wasteful). `enabled: lookupKeys.length > 0` — skips network call when no tiers requested.

9. **`SubscribeForm` prop changes — BREAKING from 0024.** Prop signature changes:
   ```ts
   // OLD (0024)
   interface SubscribeFormProps { tiers: Tier[]; onSuccess?: () => void; }

   // NEW (0024.1)
   interface SubscribeFormProps { lookupKeys: string[]; onSuccess?: () => void; }
   ```
   The `Tier` type is deleted (replaced by `TierEnvelope` from `useTiers`). SubscribeForm internally calls `useTiers(lookupKeys)` and renders from the response.

   Acceptable as a breaking change because: (a) only carddroper-staging itself ships an override of `/app/subscribe/page.tsx` today, and that ships `tiers={[]}` placeholder per the chassis-defaults pattern; (b) 0024 was committed less than a day ago — no real adopter integration relies on the old shape yet; (c) backward-compat shim adds maintenance debt with zero value.

10. **`/app/subscribe` page** — chassis ships with `<SubscribeForm lookupKeys={[]} />`. Same empty-state semantic as 0024; the chassis renders "no plans available" placeholder when `lookupKeys` is empty.

11. **No backend Subscribe-endpoint changes.** `POST /billing/subscribe { price_lookup_key, payment_method_id }` contract is unchanged. The frontend now sources `lookup_key` from the `useTiers` response instead of project-layer hardcoded Tier objects, but it still passes the same shape to the subscribe endpoint.

12. **No DB migration. No new env vars. No new dependencies.**

13. **No `chassis-contract.md` entries.** Stripe API call lookups happen at request time, not boot time. The existing `STRIPE_SECRET_KEY` invariant from 0021 covers this.

## Out of scope

- **Multi-currency support** (`BILLING_CURRENCY` stays USD-only).
- **Public pricing page** (project-layer marketing content; chassis renders inside auth-gated `/app/subscribe`).
- **Tier ordering/sorting beyond input order** — `lookupKeys` array order IS the display order; chassis preserves it.
- **Caching beyond React Query's staleTime** — no Redis, no in-memory backend cache; Stripe API rate limits + 5-minute frontend cache is sufficient.
- **Trial periods, promo codes, annual/monthly toggle UI** — already out of scope per 0024.
- **Backward-compat shim for old `tiers: Tier[]` prop** — breaking change is acceptable per Design decision #9.
- **Stripe Product image / logo rendering** — chassis ships text-only; project layer handles imagery.

**2. New endpoint `GET /billing/tiers`:**

```python
class TierEnvelope(BaseModel):
    lookup_key: str
    tier_name: str
    description: str | None
    price_display: str
    amount_cents: int
    currency: str
    interval: str
    interval_count: int
    grant_micros: int

@router.get("/tiers", response_model=list[TierEnvelope])
async def list_tiers(
    lookup_keys: str = "",  # CSV; empty → []
    user: User = Depends(get_current_user),
) -> list[TierEnvelope]:
    keys = [k.strip() for k in lookup_keys.split(",") if k.strip()]
    if not keys:
        return []

    prices = await asyncio.to_thread(
        stripe.Price.list,
        lookup_keys=keys,
        active=True,
        expand=["data.product"],
        limit=100,
    )

    envelopes: list[TierEnvelope] = []
    for price in prices.data:
        try:
            tier_name = price.metadata["tier_name"]
            grant_micros = int(price.metadata["grant_micros"])
        except (KeyError, ValueError) as exc:
            logger.warning(
                "tiers_skip_missing_metadata",
                extra={"lookup_key": price.lookup_key, "missing": str(exc)},
            )
            continue

        if price.currency.lower() != "usd":
            logger.warning(
                "tiers_non_usd_currency",
                extra={"lookup_key": price.lookup_key, "currency": price.currency},
            )

        recurring = price.recurring or {}
        envelopes.append(TierEnvelope(
            lookup_key=price.lookup_key,
            tier_name=tier_name,
            description=getattr(price.product, "description", None) if hasattr(price, "product") else None,
            price_display=format_price(
                price.unit_amount,
                price.currency,
                recurring.get("interval", "month"),
                recurring.get("interval_count", 1),
            ),
            amount_cents=price.unit_amount,
            currency=price.currency,
            interval=recurring.get("interval", "month"),
            interval_count=recurring.get("interval_count", 1),
            grant_micros=grant_micros,
        ))

    # Preserve input order. Stripe doesn't guarantee order in the response.
    by_key = {e.lookup_key: e for e in envelopes}
    return [by_key[k] for k in keys if k in by_key]
```

**1. New hook `frontend/hooks/useTiers.ts`:**

```ts
import { useQuery, type UseQueryResult } from "@tanstack/react-query";
import { api } from "@/lib/api";

export interface TierEnvelope {
  lookup_key: string;
  tier_name: string;
  description: string | null;
  price_display: string;
  amount_cents: number;
  currency: string;
  interval: string;
  interval_count: number;
  grant_micros: number;
}

export const TIERS_QUERY_KEY = ["billing", "tiers"] as const;

export function useTiers(lookupKeys: string[]): UseQueryResult<TierEnvelope[]> {
  const sortedKeys = [...lookupKeys].sort();
  return useQuery<TierEnvelope[]>({
    queryKey: [...TIERS_QUERY_KEY, sortedKeys.join(",")],
    queryFn: () => api.get<TierEnvelope[]>(
      `/billing/tiers?lookup_keys=${encodeURIComponent(sortedKeys.join(","))}`
    ),
    enabled: lookupKeys.length > 0,
    staleTime: 5 * 60_000,
    retry: false,
  });
}
```
